# O1 Online Möbius Controller

## Objective

Build a continuously running full-256 inversion organism that learns which legal
public causal events are useful instead of receiving a hand-authored inverse
score. O1 supplies bounded target-local streaming state and long-lived learned
representation; O1-O supplies deterministic action composition, contextual
failure memory and adaptive experiment selection.

The terminal objective remains exact standard twenty-round ChaCha20 verification
with all 256 key bits unknown. The first gate is lower key NLL on unseen whole-key
holdouts, not interpretability of the learned representation.

## Do not program the signal

The implementation defines only:

1. the legal public perception space;
2. the legal intervention/action space;
3. the bounded state and work budgets;
4. the truth and reward boundary.

It does not define a scalar such as “more conflicts means keybit 1.” Operator,
lane, horizon, proof-ancestry and carry features are observations. Their useful
combinations, signs, phases and timescales are learned across known-key episodes.

Nuisance removal is an invariance contract, not an inverse label. O1C-0016 shows
that global public-instance difficulty dominates h65 and survives shuffled labels.
Removing a mode shared by both polarities prevents it from masquerading as key
orientation; it does not tell the model what the remaining orientation means.

## Legal event token

Each solver/carry event is reduced online into a bounded token and discarded. A
token may contain only attacker-computable fields:

- public target-view commitment;
- key coordinate or causally eligible sparse coordinate tuple;
- tested polarity/assignment;
- requested and realized solver horizon;
- event clock, delta clock and billed work;
- operator, round, half-round, lane and bit address;
- assumption-rooted proof ancestry depth, fan-in, reuse and recurrence summaries;
- propagation, decision, conflict and closed-prefix deltas;
- SAT/UNSAT/UNKNOWN status available at that billed point;
- matched factual/control view identity;
- coverage, age and novelty state already held by the bounded controller.

No target key, target trace, candidate dictionary or growing transcript may enter
the live state. Candidate keys generated by the attacker may be evaluated and
traced, but their rows are reduced and discarded immediately.

## Boolean Möbius interventions

The degree-1 field is the exact opposite-assumption difference

```text
m_i = F(k_i=1) - F(k_i=0).
```

For a causally eligible pair, the degree-2 innovation is

```text
m_ij = F(1,1) - F(1,0) - F(0,1) + F(0,0).
```

Sparse triples may be proposed only after a pair earns positive unseen-key utility:

```text
m_ijk = F111 - F110 - F101 - F011 + F100 + F010 + F001 - F000.
```

Degree does not imply exhaustive enumeration. Eligible pairs begin with shared
carry/proof ancestry, same-addition carry neighborhoods, quarter-round coupling
and reader-disagreement edges. Coverage debt keeps every coordinate reachable;
learned utility decides where deeper interaction work is spent.

## Common-mode rejection

Before coordinate-bound accumulation, the stream exposes both raw and residual
views:

1. exact polarity difference;
2. per-target/per-horizon zero-sum centering across observed coordinates;
3. operator/lane cohort centering and robust scale normalization;
4. a bounded online low-rank nuisance model updated without key labels;
5. innovation residual, initially
   `r96 = d96 - E[d96 | d65, public nuisance state]`.

The low-rank model uses a bounded streaming update such as Oja's rule. It may learn
common amplitude and temporal modes on a live target because that update is
label-free. Any map from residual state to key polarity is slow state and cannot
be refitted on an unrevealed target.

## Fast and slow state

### Fast target-local state

Reset at each new challenge:

- GSSM nuisance and innovation state;
- 256-coordinate Bit-Vault `{logit, precision, age, coverage}` registers;
- fixed-size holographic banks addressed by coordinate, operator family and
  reader phase;
- capped sparse pair/triple cache;
- actor uncertainty, disagreement, surprise and coverage debt;
- exact resource counters.

Allowed live updates are label-free: nuisance subspace adaptation, next-event
prediction, cross-horizon consistency, antisymmetry consistency, repeatability,
novelty and contradiction reduction.

### Slow cross-challenge state

Persists while the process remains loaded:

- event encoder and residual-to-polarity reader;
- utility critic predicting key information gain per work;
- action policy and contextual failure memory;
- calibrated uncertainty and phase dictionaries;
- capped replay/Attic summaries from completed known-key episodes.

Slow weights train on self-generated known-key BUILD episodes and may consume a
sealed challenge only after its prediction was frozen, revealed and classified.
Challenge `n` may teach challenge `n+1`; it may never improve its own score.

## Learned picker

At a decision boundary O1-O deterministically composes a bounded candidate action
set. O1 scores each action using learned utility plus exploration terms:

```text
score(a) = E[delta key NLL | state, a] / predicted_work(a)
           + beta_u * uncertainty(a)
           + beta_s * surprise(a)
           + beta_c * coverage_debt(a)
           + beta_d * expert_disagreement(a).
```

The coefficients and utility representation are learned on whole-key BUILD
episodes. A small frozen exploration floor prevents starvation. No coordinate may
be assigned exact posterior mass zero solely because the policy did not inspect
it; only exact UNSAT or successful public verification can harden probability to
0 or 1.

The initial action families are:

- deepen a unary coordinate from h65 toward h96 or beyond;
- compare an alternative horizon or event family;
- open a causally linked degree-2 Möbius interaction;
- query a factual/control contrast for common-mode rejection;
- stop spending work on a contextually failed family;
- emit/update the bounded posterior and verification beam.

## Learning objectives

On self-generated known-key episodes, the primary reward is realized reduction in
true 256-bit key NLL per billed branch. Auxiliary losses may improve sample
efficiency but cannot replace that reward:

- next-event and next-horizon prediction;
- polarity-swap antisymmetry;
- factual/control contrast prediction;
- representation agreement across repeated legal views;
- uncertainty calibration;
- exact verification bonus and invalid-beam penalty.

No auxiliary objective is called inverse progress unless it improves unseen
whole-key NLL over matched controls.

## Controls

Every representation experiment retains:

- raw h65 and exact h96 sentinels;
- common-mode-only reader;
- shuffled-key training sentinel;
- frozen balanced-sign or orthogonal coordinate rotation with matched norm,
  spectrum, work and zero-sum geometry;
- output-bit flip, wrong-nonce and output-byte-rotation public views;
- fixed round-robin and uniform-random action schedulers at equal work;
- no-learning and no-interaction ablations.

O1C-0016 proves that shuffled labels alone are insufficient: the shuffled h65 fit
became a positive scalar copy of primary h65. The balanced rotation explicitly
destroys coordinate alignment while preserving amplitude geometry.

## O1C-0017 gate: learn without fresh sealed entropy

O1C-0017 generates known full-256 keys and splits by whole key and nonce. No event,
target, nonce or fitted nuisance statistic crosses folds. It reports:

- unseen-key NLL and compression;
- primary-minus-each-null NLL;
- target sign count and leave-one-key-out minimum;
- split-stable coordinates and sparse interactions;
- calibration and expected/realized information gain per branch;
- work savings against round-robin and uniform schedules;
- exact fast-state, slow-state, Attic and artifact bytes;
- counterfactual policy replay on the same frozen event pools.

Advance only if repeated folds are positive, both coordinate-destroying controls
lose, orientation is split-stable and the adaptive policy improves information per
work. Policy efficiency without NLL lift is retained as scheduler evidence, not an
inverse claim.

## O1C-0018 gate: next-challenge-only sealed test

Freeze encoder, nuisance projector, holographic addresses, reader, policy,
thresholds, budgets, controls and exact posterior serialization before fresh
entropy. Then attack entirely new standard full-round keys under the public-only
contract. All predictions persist before reveal. O1C-0015 and O1C-0016 targets are
never reused.

If O1C-0018 lowers NLL, the residual field may enter a bounded verification beam
and causal Attic. If O1C-0017 already fails known-key holdouts, no sealed panel is
spent: global proof difficulty is closed and the sensor pivots directly to
query-rooted carry/proof ancestry.

## Continuous runtime

The production form is one long-lived CPU process:

```text
load slow model once
for challenge in stream:
    snapshot slow-model commitment
    reset bounded target-local fast state
    perceive -> choose -> intervene -> update until work/stop boundary
    persist posterior and exact resource receipt
    reveal/verify only through the challenge boundary
    classify the frozen attempt
    update slow model for the next challenge
```

Separate run capsules are immutable challenge epochs, not process restarts. They
provide clean target boundaries and a readable research cockpit while the learned
organism itself can remain resident and improve continuously.
